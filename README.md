Конкретно в твоём случае агент делает одну вещь — критикует резюме как опытный HR из ML-компании.
Вот что происходит когда ты нажимаешь «Анализировать»:
Шаг 1 — сбор данных
JS берёт твой текст резюме + выбранную роль (например «ML Engineer») и склеивает их в один большой промпт.
Шаг 2 — запрос к модели
Этот промпт улетает в Groq, там Llama 3.3 70B его читает. Модель получает инструкцию: «ты строгий ревьюер, проанализируй по 5 секциям, верни JSON».
Шаг 3 — анализ
Модель проходится по каждой секции резюме и оценивает:

Навыки — достаточно ли конкретики, есть ли нужный стек для роли
Проекты — есть ли метрики, понятен ли результат
Опыт — сильные ли глаголы, не расплывчато ли написано
Образование/курсы — релевантны ли они роли
Оформление — структура, читаемость

Шаг 4 — результат
Модель возвращает JSON, JS его разбирает и рисует карточки: зелёные (ок), жёлтые (есть проблемы), красные (слабо). Для слабых секций сразу показывает как переписать конкретные фразы.
Главная польза — модель видит резюме глазами рекрутера и говорит не «неплохо», а «вот эта фраза слабая, вот как надо написать».
